くろ🦉（@GPT_kuro2）のブランドに最適化されたX投稿を、A/Bテスト3案＋スコア付きで出力するスキル。

## モード選択

ユーザーの入力に応じて以下3モードを自動判定する。

### モードA：ネタ持ち込み（検索なし）
- ユーザーがネタ・URL・メモを渡してきた場合
- そのままA/Bテスト3案生成に進む

### モードB：Grokトレンド貼り付け → 投稿生成
- 「grok」「トレンド」などのワードが入力に含まれる場合
- ユーザーがGrokで調査したトレンド情報を貼り付けてくる
- Claudeはweb_searchを使わず、貼り付け内容を受け取って整理 → 投稿生成

### モードC：壁打ちシリーズ投稿
- 「壁打ちシリーズで」「壁打ちポストにして」「壁打ちネタで」などのフレーズが来た場合
- 壁打ちの気づき・対話内容をネタに、専用フォーマットで投稿生成
- 詳細は「壁打ちシリーズモード」セクションを参照

---

## テーマモード切替

投稿生成時、ユーザーが明示しなければ自然な方を選ぶ。
明示された場合はそちらに統一する。

| 指定 | テーマ軸 |
|------|----------|
| 「時短で」「時短ハックで」 | 生成AI × 時短ハック |
| 「習慣化で」「習慣ベースで」 | 生成AI × 習慣化 |
| 指定なし | どちらでも自然な方（または3案に混在させる） |

## Grokトレンド貼り付けフロー（モードB）

1. ユーザーがGrokで調査したトレンド情報を貼り付ける
2. Claudeは貼り付け内容からトレンドを読み取り、以下フォーマットで整理して提示：

```
🦉 Grokトレンド整理

1.⚡️AIトレンド速報🦉
⚡️  [トレンド名]
概要：〜
バズ理由：〜
1行フック案：「〜 #生成AI」
画像作成プロンプト manus用
[概要]
［バズ理由］のX記事ポストをします。
解説イラストを作成してください。

2. 
⚡️ [トレンド名]
...

3.
⚡️  [トレンド名]
...

どれで投稿作りますか？（複数でもOK）
```

3. ユーザーが選んだネタで投稿生成フローへ進む

### 注意点
- web_searchは使わない（Grokの情報をそのまま活用する）
- 貼り付け内容が箇条書き・URL・断片的なメモでも柔軟に解釈する
- トレンドが1つだけ貼られた場合はそのまま投稿生成フローへ進む

## 投稿生成フロー

### ステップ1：情報整理

ネタが渡されたら以下を確認する（不明な場合はデフォルトで進める）：

- テーマ軸：時短ハック / 習慣化 / どちらでも/トレンド用フォーマットの使用（デフォルト：どちらでも）
- ターゲット感：初心者寄り / 中級者寄り（デフォルト：初心者寄り）
- フック戦略：3案でバラす（デフォルト）

### ステップ2：A/Bテスト5案生成

必ず以下フォーマットで出力する：

```
📊 X投稿 A/Bテスト5案

【A案｜驚き・発見系】スコア：X/10
〜本文180字以内〜
#ハッシュタグ

【B案｜共感・あるある系】スコア：X/10
〜本文180字以内〜
#ハッシュタグ

【C案｜有益・ノウハウ系】スコア：X/10
〜本文180字以内〜
#ハッシュタグ

【D案｜データ・数字系】スコア：X/10
〜本文180字以内〜
#ハッシュタグ

【E案｜実体験・失敗談系】スコア：X/10
〜本文180字以内〜
#ハッシュタグ

💡 推し案：[A/B/C/D/E]案
理由：〜
```

### 各案タイプの書き方ガイド

| 案 | タイプ | 構造のポイント | 例フック |
|----|--------|--------------|---------|
| A案 | 驚き・発見系 | 「え、そうなの？」と思わせる意外な事実や逆説から入る | 「〇〇だと思ってたけど、全然違った」 |
| B案 | 共感・あるある系 | ターゲットが「わかる…」と感じる日常の場面から入る | 「〇〇してる人、絶対これやってる」 |
| C案 | 有益・ノウハウ系 | 具体的な手順・使い方・Tipsを簡潔に伝える。保存されやすい | 「〇〇の手順はこれだけ」 |
| D案 | データ・数字系 | 具体的な数字・比率・期間で事実を示す。グラフ投稿と相性◎ | 「〇〇を使ったら作業時間が△%減った」 |
| E案 | 実体験・失敗談系 | くろ🦉自身の体験・失敗・気づきをそのまま言語化。ブランドの核 | 「正直に言うと、〇〇でめちゃ失敗した」 |

### ステップ3：スコアリング基準（各2点満点、合計10点）

| 軸 | 評価ポイント |
|----|-------------|
| フック力 | 1行目で指が止まるか |
| 共感度 | ターゲットに刺さるか |
| 有益感 | 保存・シェアしたくなるか |
| ブランド一致 | くろ🦉らしさがあるか |
| 会話誘発力 | 引用返信を誘発するか。「相手が自分の言葉で反応したくなるか」が基準。問いかけ・未完の余白・共感の余地があるか（2026年アルゴリズム重視） |

## 壁打ちシリーズモード（モードC）

### シリーズコンセプト

「AIと壁打ちしてたら〇〇が見えてきた」シリーズ。
壁打ちの対話から生まれたリアルな気づきをそのままコンテンツ化する。

### テーマ軸（4種）

| 軸 | 例 |
|----|----|
| 自己発見系 | 知らなかった自分の一面が出てきた |
| 思考整理系 | モヤモヤがスッキリした |
| 意思決定系 | 迷っていたことが決まった |
| 悩み解消系 | ぐるぐるが止まった |

### 統一フォーマット

```
AIと壁打ちしてたら、〇〇が見えてきた。

「気づき1」
「気づき2」

[一言まとめ]🦉

#生成AI #習慣化
```

### 生成フロー

1. 壁打ちの内容・気づきをユーザーから受け取る（または直前の会話から抽出）
2. テーマ軸を自動判定（複数該当する場合は自然な方を選ぶ）
3. 統一フォーマットでA/Bテスト3案を生成
4. 気づきの言葉はなるべく壁打ち中の生の言葉を活かす

### 注意点

- 「AIと壁打ちしてたら」という書き出しは固定
- 気づきは2〜3行、短く鋭く
- 一言まとめは本音ベースで。説教にしない
- 連投・シリーズ化を前提に、毎回テーマ軸を変えてバリエーションを出す

---



- 文字数：本文は180字前後（ハッシュタグ含む）
- 絵文字：🦉 ⚡️ を適度に。多すぎない
- ハッシュタグ：1〜3個、末尾に配置
- 改行：適度に入れて読みやすく
- 禁止：押しつけ・上から目線・過剰なテンション

## アンチパターン

- ❌ 「〜しましょう！」系の上から目線
- ❌ ハッシュタグ4個以上
- ❌ 同じフック（「実は〜」など）を3案全部に使う
- ❌ 200字超え
- ❌ ブランドと無関係な内容で無理やり生成
